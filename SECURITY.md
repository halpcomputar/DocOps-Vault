# Security Policy

DocOps Vault is primarily a Markdown starter and workflow template, but security and privacy issues can still happen.

## Report Privately When Possible

Do not post secrets, private client data, real incident details, screenshots, logs, domains, IPs, or credentials in public issues.

If GitHub private vulnerability reporting is enabled, use it.

If private reporting is not available, open a minimal public issue that describes the category of problem without sensitive details and ask for a private contact path.

## What Counts

Security or privacy issues may include:

- accidental inclusion of private data in this repository
- templates or prompts that encourage unsafe handling of secrets
- scripts or workflows that expose private files
- unsafe GitHub Actions or automation behavior
- guidance that could cause users to publish raw source material accidentally

## Scope

This project does not provide professional security, legal, compliance, or incident-response advice. It is a documentation starter. Users are responsible for validating workflows against their own operational, legal, and privacy requirements.
