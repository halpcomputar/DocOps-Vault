# Incident Intake

Use this folder when you already know a batch of raw material belongs to an incident.

Recommended shape:

`raw/incidents/<client-slug>/YYYY-MM-DD-<incident-slug>/`

Files inside can be loose or grouped by source type:

- `screenshots/`
- `emails/`
- `reports/`
- `logs/`
- `recordings/`
- `notes/`

This is an intake shortcut, not a new media type. Normal source folders such as `raw/screenshots/`, `raw/reports/`, and `raw/recordings/` can still feed incident evidence when their source-specific ingest pipeline detects incident relevance.
